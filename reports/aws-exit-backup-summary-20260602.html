5. 복원할 때 먼저 볼 것

문서/구조 복원

  1. GitHub repo를 clone합니다.
  2. README.md, restore/restore-runbook.md, docs/local-portability.md를 확인합니다.
  3. 민감값은 GitHub가 아니라 새 환경의 Secret Manager/환경변수에 다시 주입합니다.

대용량 artifact 복원

  1. Google Drive 백업 루트에서 필요한 artifact만 내려받습니다.
  2. S3/RAG PDF/model artifact/Lambda zip/Docker image를 새 실행 환경에 맞게 배치합니다.
  3. Qwen은 skopeo로 OCI layout에서 새 registry로 복사합니다.

Qwen 복원 예시

brew install skopeo
rclone copy gdrive:3team-aws-exit-backup-20260602/docker-images/qwen-oci-layout ./qwen-oci-layout
skopeo copy oci:./qwen-oci-layout:amd64-vllm-r4-awq32k docker://<new-registry>/sagemaker-qwen-vllm-v3:amd64-vllm-r4-awq32k

6. 남은 리스크 / 추가로 해야 할 수 있는 일

가장 큰 리스크: RDS 실제 데이터 dump 미확보

로컬에서 RDS endpoint 접근이 되지 않아 실제 PostgreSQL dump는 성공하지 못했습니다. AWS 계정 삭제 전 RDS 데이터가 중요하면 VPC 내부에서 dump 또는 snapshot export/share가 필요합니다.

  • RDS: VPC 내부 ECS/Lambda/bastion에서 pg_dump, 또는 snapshot export to S3, 또는 snapshot 타 계정 공유/복사 필요.
  • 로컬 이식: RAG는 RAG_PDF와 Drive의 s3/say2-3team/rag-pdfs/ 기반으로 재구축 가능성이 높습니다.
  • 비밀값: 기존 AWS secret 값은 문서화하지 않았으므로 새 환경에서 별도 재발급/재입력해야 합니다.
